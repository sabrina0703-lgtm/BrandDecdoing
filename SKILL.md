# BrandDecoding — 品牌生意解码工作流

## 概述

本技能编码一套标准化的品牌生意解码工作流（BrandDecoding）。当用户输入品牌名后，按7大模块分步深度解码该品牌的生意现状，每个模块完成后用户审核确认再推进，最终生成一份LV高级商业风格的交互式HTML网页报告。

**核心原则（来自实战经验，必须遵守）**:
1. **数据来源真实权威** — 每条数据标注来源和时间，估算数据明确标注"估算"
2. **分析导向，而非数据堆砌** — 每个章节必须有分析归纳和mini-conclusion，不是零散信息的集合
3. **Key finding必须附具体数值** — 任何关键论断都要有数字支撑，不允许模糊表述
4. **严格对齐用户框架** — 只使用用户brief的7模块模型，不自行增加或减少分析维度
5. **分步确认，逐步推进** — 不跳过用户审核
6. **数据来源：联网搜索 + 用户手动提供，两者结合**
7. **所有外币金额换算为人民币呈现**

## 一、启动协议

### 1.1 触发条件
当用户表达以下意图时触发：
- "分析XX品牌"
- "帮我分析XX品牌的生意"
- "XX品牌怎么样"
- "品牌分析 XX"
- 用户主动提供PDF/图片等资料时也触发（作为补充数据源）

### 1.2 启动流程

**Step 1: 品牌确认**
```
收到品牌名后:
1. 联网快速搜索该品牌的基本信息（所属公司、行业、品类）
2. 向用户确认:
   "确认分析品牌: 【品牌名】
    所属公司: 【公司名】
    所属行业/品类: 【推测的行业品类】
    品牌类型: 【中国本土/跨国】

    以上信息是否正确？请确认或修正。
    您是否有内部资料（PDF/图片等）需要提供？"
```

**Step 2: 工作流说明**
```
"好的，开始分析【品牌名】。

 工作流分为7个模块:
 1. 宏观环境分析 (PEST)
 2. 行业洞察
 3. 品牌洞察 (4P框架)
 4. 用户洞察
 5. 竞争格局
 6. 财务与增长

```

## 二、用户资料处理

如果用户提供了PDF/PPT/图片等文件：
1. **纯文字PDF**: 直接用 pymupdf get_text() 提取
2. **纯图片PDF/PPT**: 安装 `rapidocr-onnxruntime`（pip install），用OCR提取文字。脚本参考：
   ```python
   from rapidocr_onnxruntime import RapidOCR
   ocr = RapidOCR()
   result, elapse = ocr(img_path)
   # result = [[bbox, text, confidence], ...]
   ```
3. **单张图片**: 直接OCR处理
4. 提取后保存为 txt 文件供后续分析使用
5. OCR提取的内容标注为"品牌方内部资料"，与公开数据区分

## 三、分步确认机制

每个模块完成后，展示分析摘要并询问用户：

```
【模块X】分析完成，摘要如下:
[展示该模块的核心发现、关键数据、分析结论]
[标注所有数据来源]

您可以:
1. 回复"确认"或"继续" — 进入下一模块
2. 回复"修改: [具体内容]" — 调整分析
3. 回复"补充: [数据/信息]" — 添加额外数据
4. 回复"跳过" — 标记为数据不足，继续下一模块
```

## 四、7大模块详细工作流

**详细数据采集清单和分析框架见 `references/framework.md`**

### 模块1: 宏观环境分析 (PEST)
- 搜索关键词模板见 references/framework.md
- 四因素（政策/经济/社会/技术）逐项分析
- 输出：PEST因子分析卡片 + 小结结论（一句话宏观结论）

### 模块2: 行业洞察
- TAM-SAM-SOM三层市场模型
- 行业生命周期判断
- 波特五力模型
- CR5集中度 + 行业KSF
- **CR5计算方法必须说明**：CR5 = 前5名市场份额之和，并对照贝恩分类法标准（>75%为寡占型）
- 输出：市场规模表 + 生命周期判断 + 波特五力评分 + CR5排名 + 一句话赛道判断

### 模块3: 品牌洞察 (4P框架)
- 品牌定位（Positioning）: UVP提炼 + 品牌金字塔 + 生态位 + 品类生命周期 + 定位锋利度
- 产品矩阵: BCG四象限图（**用SVG象限图呈现**，不是表格）
- 价格锚定分析
- 渠道ROI矩阵
- 营销效率漏斗
- **品牌生态位用SVG图表呈现**（价格带×品类宽度战略定位图），不是纯文字
- 输出：定位一句话总结 + 品牌金字塔 + BCG象限图 + 价格对标表 + 渠道权重表 + 生态位图

### 模块4: 用户洞察
- 用户画像五维模型
- 5A模型 (科特勒)
- 消费者旅程地图CJM
- JTBD分析
- 用户需求金字塔
- 输出：核心用户画像卡片 + 5A评估表 + CJM地图 + 品牌心智关键词 + 痛点清单

### 模块5: 竞争格局
- 竞品三级分层（Tier1/2/3）
- 竞品雷达图（7维1-5分）
- 战略群组图（价格带×品类覆盖度）
- 输出：竞品全景图 + 核心竞品拆解 + 雷达图 + 战略群组图 + 差异化对比表 + 一句话竞争结论

### 模块6: 财务与增长
- 商业模式画布（9要素）— **放在报告最前面作为品牌画像Preview**
- 增长三要素拆解（量增×价增+新品类）
- S曲线增长模型
- UE模型（LTV/CAC）
- 飞轮效应
- 上市公司财报分析（如适用）
- **所有外币金额换算为人民币**（汇率标注来源）
- 输出：商业模式画布 + 营收估算表 + 增长拆解 + S曲线 + UE估算 + 飞轮模型 + 一句话财务结论

### 模块7: 综合诊断
- SWOT战略矩阵（SO/WO/ST/WT四象限交叉策略）
- 三句话核心结论
- P0/P1/P2策略建议（含具体动作+预期效果+优先级理由）
- 商业模式健康度5维评分
- 风险预警清单
- 输出：SWOT矩阵 + 三句话结论 + P0/P1/P2卡片 + 5维评分表 + 风险清单 + 品牌评级

## 五、报告生成（HTML）

### 5.1 输出格式
**HTML网页报告**（不是Word），用户可直接在浏览器中查看。

### 5.2 视觉风格
LV高级商业风格，具体CSS见 `assets/report-template.html`：
- 色板: 深棕#3E2D1C + 象牙白#F5F0E8 + 金色#C5A572
- 字体: Cormorant Garamond（英文标题/衬线） + Noto Serif SC（中文正文/思源宋体），通过Google Fonts加载
- 直角线条（无圆角），极简1px细线边框
- 金色accent用于章节顶线、SWOT色条、策略卡片标签

### 5.3 报告结构

```
左侧固定导航栏（深棕底+金色分割线，滚动高亮当前章节，点击平滑跳转）

封面（深棕底色+金色内框+大字间距UPPERCASE）

品牌画像 Preview（商业模式画布9要素）

【品牌名】现状概览 Summary（9张信息卡片）:
  生意规模 | 增速 | 市场占有率（整体份额+细分赛道份额）
  核心用户 | 城市 | 渠道 | 平均价格 | 核心产品 | 品牌主张

目录（金色分割线，点击跳转）

模块1：宏观环境分析（PEST）
模块2：行业洞察
模块3：品牌洞察（4P框架）
模块4：用户洞察
模块5：竞争格局
模块6：财务与增长
模块7：综合诊断（SWOT + P0/P1/P2）

附录：数据来源清单
```

### 5.4 每个模块的写作规范（关键！）

1. **分析导向**: 先写分析叙述（归纳、对比、判断），再呈现支撑数据表格
2. **Mini-conclusion**: 每个模块末尾必须有蓝色底金色左框的"本节小结"框
3. **Key finding附数值**: 任何关键论断用具体数字支撑，不允许"份额超过XX"之类的模糊表述
4. **数据来源标注**: 每个表格下方标注来源和时间
5. **BCG矩阵用SVG象限图**: 不是表格
6. **品牌生态位用SVG定位图**: 不是纯文字
7. **SWOT用四宫格**: 绿/红/蓝/灰配色+左3px色条
8. **策略卡片**: P0红/P1棕/P2绿，1px细线边框+UPPERCASE衬线标签

### 5.5 生成流程

1. 将7模块分析结果整理为结构化内容
2. 读取 `assets/report-template.html` 获取CSS和HTML结构模板
3. 基于模板+分析内容，编写完整的HTML报告
4. 保存为 `【品牌名】_品牌生意深度分析报告.html`
5. 用 present_files 展示给用户

### 5.6 侧边栏导航JS

报告必须包含自动生成导航+滚动高亮的JavaScript。代码见 `assets/report-template.html` 底部的 `<script>` 部分。

## 六、跨国品牌 vs 中国本土品牌差异

执行分析时，根据品牌类型调整数据来源：

| 维度 | 中国本土品牌 | 跨国品牌追加 |
|------|-------------|-------------|
| 政策 | 国内产业政策 | +国际贸易政策、关税、地缘政治 |
| 经济 | 国内宏观指标 | +全球/区域经济指标 |
| 行业 | 中国市场规模 | +全球市场规模、区域分布 |
| 渠道 | 天猫/京东/抖音 | +亚马逊/Shopee/Lazada |
| 财务 | 国内上市/融资 | +SEC/港交所/DART等披露 |
| 用户 | 中国消费者特征 | +跨文化消费者差异 |
| 搜索 | 中文为主 | +中英文双语搜索 |
| 额外源 | — | +Statista、Euromonitor Global、World Bank、IMF |
| 货币 | 人民币 | +外币换算为人民币（标注汇率来源） |

## 七、关键数据公式

```
TAM = 目标用户总数 × 人均消费金额 × 消费频次
SAM = TAM × 品牌可达比例
SOM = SAM × 品牌实际市占率
CAGR = (终值/初值)^(1/年数) - 1
CR5 = 前5名市场份额之和（>75%=寡占型·贝恩分类法）
HHI = Σ(各品牌市占率²) × 10000  // >2500为高度集中
LTV = 平均客单价 × 年均购买次数 × 客户生命周期(年) × 毛利率
UE健康标准 = LTV / CAC > 3
外币→RMB换算: 标注汇率来源和换算日期
```

## 八、数据可信度分级

所有引用数据标注来源等级：
- **Tier 1 (官方一手)**: 公司IR页面、DART/SEC备案、国家统计局
- **Tier 2 (监管/权威二手)**: 行业协会白皮书、上市券商研报
- **Tier 3 (媒体/第三方)**: 新闻报道、咨询公司公开摘要、电商平台数据

交叉验证规则：重要财务数据至少用Tier 1+Tier 2两源对照，差异>5%时标注说明。
